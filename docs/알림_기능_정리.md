# With:溫 앱 — 알림 기능 정리

## 1. 구현된 알림 기능 목록

| 구분 | 기능 | 설명 | 사용 패키지/기술 |
|------|------|------|------------------|
| **1** | **매일 기도 알림 (로컬)** | 사용자가 설정한 시간에 매일 "오늘의 기도를 나눌 시간이에요" 알림 | `flutter_local_notifications`, `timezone`, `flutter_timezone`, `shared_preferences` |
| **2** | **알림 설정 UI** | 나의 발자국 > 기도 알림 설정: ON/OFF 토글, 알림 시간 선택 | 프로필 페이지 내 카드 UI |
| **3** | **알림 권한 요청** | Android 13+ / iOS 알림 권한 요청 | `NotificationService.requestPermissions()` |
| **4** | **설정 유지** | 알림 ON/OFF·시간을 로컬에 저장, 앱 재실행 시 복원 및 재스케줄 | `SharedPreferences`, `MainScaffold.initState` |
| **5** | **FCM 토큰 저장** | 로그인 후 FCM 토큰을 Firestore `users/{uid}.fcm_token`에 저장·갱신 | `firebase_messaging`, `cloud_firestore` |
| **6** | **FCM 수신 준비** | 백그라운드 메시지 핸들러 등록, 포그라운드/알림 탭 수신 리스너 | `firebase_messaging` |

| 환경 | 로컬 알림 (매일 기도) | FCM 수신 |
|------|------------------------|----------|

## 3. Firebase Spark(무료) 플랜에서 사용 불가인 알림 기능

Firebase **Spark 플랜**에서는 아래 기능을 **서버 측에서** 사용할 수 없습니다.

| 기능 | 이유 | Spark에서 불가인 부분 |
|------|------|------------------------|
| **소그룹 실시간 푸시 (FCM 발송)** | 푸시를 **보내는** 쪽이 서버(Cloud Functions 등)에 있음 | **Cloud Functions** 배포 불가 → `group_prayers` 문서 생성 시 다른 멤버에게 FCM 전송하는 로직을 Spark에서 실행할 수 없음 |

- **FCM 클라이언트(앱)**: 토큰 받기, 토큰 Firestore 저장, 메시지 **수신** → Spark에서도 동작합니다.
- **FCM 서버(메시지 발송)**: “누군가 기도 제목을 공유했을 때 푸시 보내기”처럼 **이벤트에 반응해 메시지를 보내는** 부분은 Cloud Functions(또는 다른 백엔드)가 필요하고, **Cloud Functions는 Blaze 플랜에서만** 배포·실행 가능합니다.

---

## 4. Spark 플랜에서 사용 가능한 알림 기능 요약

- ✅ **매일 기도 알림 (로컬)**: 기기에서만 동작하므로 Firebase 유료 기능 없이 사용 가능.
- ✅ **알림 설정 UI, 권한 요청, 설정 유지, 재스케줄**: 모두 클라이언트만 사용.
- ✅ **FCM 토큰 저장**: Firestore 쓰기만 하므로 Spark 한도 내에서 가능.
- ✅ **FCM 메시지 수신**: 다른 도구(Blaze에서 배포한 Cloud Functions, 테스트용 콘솔 등)에서 보낸 메시지는 Spark 앱에서도 수신 가능.

---

## 5. 소그룹 푸시를 쓰려면 (Blaze 전환 시)

1. Firebase 프로젝트를 **Blaze(종량제)** 로 전환.
2. **Cloud Functions** 배포: `group_prayers` 문서 생성 시 같은 그룹 멤버의 `fcm_token`으로 FCM 발송.
3. (선택) iOS의 경우 Firebase Console에서 **APNs 인증 키** 설정.

현재 프로젝트의 `functions/index.js`는 기본 템플릿 상태이며, 위와 같은 FCM 발송 함수는 **아직 포함되어 있지 않습니다.**  
과거에 제안된 `onGroupPrayerShared` 형태의 함수를 사용하려면 Blaze 전환 후 해당 코드를 추가해 배포해야 합니다.
